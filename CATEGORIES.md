# Respec Skill Categories

Official taxonomy for organizing and discovering skills.

## Categories

### `frontend`
React, Vue, Next.js, UI libraries, component systems

**Examples:** nextjs, shadcn, frontend-design

---

### `backend`
APIs, databases, server architecture, Node.js, Go, Python servers

**Examples:** architecture-patterns, prisma, sql-expert

---

### `fullstack`
End-to-end patterns spanning frontend + backend

**Examples:** nextjs + prisma combos, MERN stack

---

### `testing`
Unit, integration, E2E testing, test infrastructure

**Examples:** javascript-testing-patterns, senior-qa

---

### `deployment`
CI/CD, containers, cloud platforms, infrastructure as code

**Examples:** docker-containerization

---

### `ai`
LLM apps, RAG, agents, embeddings, ML workflows

**Examples:** llm-app-patterns, llm-security, senior-prompt-engineer

---

### `data`
SQL, data pipelines, analytics, ETL

**Examples:** sql-expert, senior-data-scientist

---

### `mobile`
React Native, Flutter, native iOS/Android

**Examples:** (none currently)

---

### `design`
UI/UX, component systems, accessibility, visual design

**Examples:** adapt, animate, audit, frontend-design, teach-impeccable

---

### `devops`
Infrastructure, monitoring, automation, observability

**Examples:** healthcheck, git-advanced-workflows

---

### `security`
Auth, encryption, vulnerability scanning, secure coding

**Examples:** software-security, llm-security

---

### `productivity`
Tooling, workflows, project management, automation

**Examples:** skill-creator, skill-evaluator, nightly, brain-dump-assistant

---

### `specialized`
Domain-specific or cross-cutting concerns

**Examples:** gastown, mcp-builder, docs-management

---

## Category Assignment Rules

1. **Single category per skill** (primary category)
2. **Choose the most specific category** (frontend > fullstack)
3. **When unclear, choose by primary use case** (nextjs = frontend, even though it has backend)
4. **Specialized is the catch-all** for unique skills

## Adding New Categories

New categories require:
- At least 5 skills that fit
- Clear differentiation from existing categories
- Community consensus (proposal via GitHub issue)
